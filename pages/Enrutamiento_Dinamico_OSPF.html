Enrutamiento Dinámico OSPF

Configuración avanzada en área única (Área Backbone 0) con 4 Routers.

1. ¿Qué es el Enrutamiento Dinámico OSPF?

OSPF (Open Shortest Path First) es un protocolo de enrutamiento de estado de enlace a nivel de redes Interiores (IGP). A diferencia de RIPv2, OSPF no cuenta saltos; en su lugar, utiliza el algoritmo de Dijkstra para calcular el camino más corto y rápido basándose en el ancho de banda (costo) de los enlaces.

Para configurarlo adecuadamente en Cisco, es crucial comprender dos conceptos fundamentales:

  • Máscara Wildcard: Es el inverso de la máscara de subred tradicional. Se obtiene restando la máscara habitual a 255.255.255.255 (ej. para una máscara /24 o 255.255.255.0, su wildcard es 0.0.0.255).
  • Áreas OSPF: OSPF divide la red en áreas para segmentar el tráfico. En topologías pequeñas o medianas se utiliza obligatoriamente el Área 0 (también llamada área backbone o troncal).

2. Elementos a utilizar en Cisco Packet Tracer

Para construir esta infraestructura en malla cuadrada, utilizaremos:

  • 4 Routers Cisco 2911: Actuarán como nodos principales (Router 0, Router 1, Router 2 y Router 3).
  • 4 Switches Cisco 2960-24TT: Uno para distribuir la señal cableada en cada sección local.
  • 4 Computadoras (PC): Un host final por red (PC0, PC1, PC2 y PC3) para realizar las pruebas de convergencia.
  • Cable Copper Cross-Over (Cable Cruzado): Para realizar los 4 enlaces WAN consecutivos de router a router.
  • Cable Copper Straight-Through (Cable Directo): Para interconectar los dispositivos finales y los switches.

3. Diseño de la Red y Direccionamiento

La topología cierra un anillo perimetral donde cada esquina representa una red LAN local independiente:

Redes Locales (LAN)

  • LAN Router 0: 192.168.10.0 /24
  • LAN Router 1: 192.168.20.0 /24
  • LAN Router 2: 192.168.30.0 /24
  • LAN Router 3: 192.168.40.0 /24

Enlaces WAN (Malla Cuadrada)

  • Superior (R0 ↔ R1): 192.168.15.0 /24
  • Derecho (R1 ↔ R2): 192.168.25.0 /24
  • Inferior (R3 ↔ R2): 192.168.35.0 /24
  • Izquierdo (R0 ↔ R3): 192.168.45.0 /24

4. Configuración del Enrutamiento Dinámico OSPF

En OSPF declaramos las redes directamente conectadas utilizando un identificador de proceso (Process ID) y la máscara wildcard, asignándolas todas al área 0.

Comandos para Router 0 (Ejemplo):

Router 0 está directamente conectado a su propia LAN (10.0), al enlace Superior (15.0) y al enlace Izquierdo (45.0).

Router_0> enable
Router_0# configure terminal

! Iniciamos el proceso OSPF con ID 1
Router_0(config)# router ospf 1

! Anunciamos las redes con su Wildcard (0.0.0.255) en el Área 0
Router_0(config-router)# network 192.168.10.0 0.0.0.255 area 0
Router_0(config-router)# network 192.168.15.0 0.0.0.255 area 0
Router_0(config-router)# network 192.168.45.0 0.0.0.255 area 0
Router_0(config-router)# end
Router_0# write memory

Redes a anunciar en el resto de los Routers:

Debes ingresar a router ospf 1 en cada equipo y registrar sus 3 redes vecinas correspondientes:

  • Router 1: Redes 192.168.20.0, 192.168.15.0 y 192.168.25.0
  • Router 2: Redes 192.168.30.0, 192.168.25.0 y 192.168.35.0
  • Router 3: Redes 192.168.40.0, 192.168.35.0 y 192.168.45.0

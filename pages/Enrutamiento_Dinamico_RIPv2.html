Enrutamiento Dinámico RIPv2

Configuración y automatización de una topología en anillo con 4 Routers.

1. ¿Qué es el Enrutamiento Dinámico RIPv2?

El enrutamiento dinámico permite que los routers intercambien información sobre la estructura de la red de forma automática. RIPv2 (Routing Information Protocol v2) es un protocolo de vector de distancia que utiliza el conteo de saltos como métrica para determinar el mejor camino (con un máximo de 15 saltos).

A diferencia de su versión 1, RIPv2 es un protocolo sin clase (classless), lo que significa que envía la máscara de subred en sus actualizaciones, soporta VLSM y utiliza la dirección multicast 224.0.0.9 para comunicarse con sus vecinos de forma eficiente.

2. Elementos a utilizar en Cisco Packet Tracer

Para armar este escenario de 4 redes en anillo necesitaremos:

  • 4 Routers Cisco 2911: Uno para el núcleo de cada subred LAN.
  • 4 Switches Cisco 2960: Para interconectar las terminales de cada red local.
  • 4 Computadoras (PC): Un host final asignado a cada LAN para pruebas de ping.
  • Cable Copper Cross-Over (Cable Cruzado): Esencial para conectar los routers entre sí consecutivamente formando el anillo central.
  • Cable Copper Straight-Through (Cable Directo): Para interconectar las PCs a los switches y estos a los routers.

3. Diseño de la Red y Direccionamiento

La topología distribuye de manera uniforme los segmentos IP tanto para los usuarios locales como para los tramos inter-router:

Redes Locales (LAN)

  • Red 1: 192.168.100.0 /24 (GW: .1)
  • Red 2: 192.168.150.0 /24 (GW: .1)
  • Red 3: 192.168.200.0 /24 (GW: .1)
  • Red 4: 192.168.250.0 /24 (GW: .1)

Enlaces WAN (Anillo)

  • Enlace R1 ↔ R2: 192.168.120.0 /24
  • Enlace R2 ↔ R3: 192.168.125.0 /24
  • Enlace R3 ↔ R4: 192.168.130.0 /24
  • Enlace R4 ↔ R1: 192.168.135.0 /24

4. Explicación y Comandos de Enrutamiento RIPv2

En el enrutamiento dinámico RIPv2, no configuramos rutas hacia destinos lejanos. En su lugar, entramos al protocolo y declaramos mediante el comando network únicamente las redes que el router tiene directamente conectadas. Él se encargará de compartirlas con sus vecinos.

Ejemplo de configuración en el Router 1 (R1):

R1 tiene conectadas tres redes: su propia LAN (100.0) y los enlaces hacia R2 (120.0) y R4 (135.0).

Router_1> enable
Router_1# configure terminal

! 1. Activamos el protocolo RIP
Router_1(config)# router rip

! 2. Cambiamos a la versión 2 (Obligatorio)
Router_1(config-router)# version 2

! 3. Desactivamos la sumarización automática de clases
Router_1(config-router)# no auto-summary

! 4. Declaramos sus redes directamente conectadas
Router_1(config-router)# network 192.168.100.0
Router_1(config-router)# network 192.168.120.0
Router_1(config-router)# network 192.168.135.0
Router_1(config-router)# end
Router_1# write memory

*Nota: Debes aplicar la misma lógica en los Routers 2, 3 y 4 utilizando las redes que les correspondan según sus conexiones físicas.

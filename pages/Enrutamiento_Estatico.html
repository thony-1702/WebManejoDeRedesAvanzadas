Enrutamiento Estático

Configuración de una topología de tres redes interconectadas.

1. ¿Qué es el enrutamiento estático?

El enrutamiento estático es un método de configuración de redes donde el administrador de la red ingresa manualmente las rutas en la tabla de enrutamiento del router. A diferencia de los protocolos dinámicos (que descubren rutas automáticamente), aquí le indicamos explícitamente al router por qué puerto o hacia qué IP debe enviar la información para alcanzar una red de destino que no tiene conectada directamente.

2. Elementos a utilizar en Cisco Packet Tracer

Para construir esta topología, arrastraremos al área de trabajo los siguientes componentes:

  • 3 Routers 2911: Servirán como las puertas de enlace de cada red y enrutarán el tráfico entre ellas.
  • 3 Switches 2960: Para conectar las computadoras de cada red local hacia su respectivo router.
  • 9 PCs (Desktop): Tres computadoras para la Red A, tres para la Red B y tres para la Red C.
  • Cable Copper Straight-Through (Cable Directo): Para conectar las PCs a los switches, y los switches a los puertos Gigabit de los routers.
  • Cable Copper Cross-Over (Cable Cruzado): Para conectar los routers entre sí (puerto a puerto) creando el anillo central.

3. Diseño de la Red y Direccionamiento

Redes Locales (LAN)

Cada router usará la primera IP disponible (.1) de su red como puerta de enlace.

  • Red A: 192.168.10.0 /24
  • Red B: 192.168.20.0 /24
  • Red C: 192.168.30.0 /24

Enlaces entre Routers

Conexiones consecutivas usando cables cruzados.

  • Router A ↔ Router B: 192.168.40.0 /24
  • Router B ↔ Router C: 192.168.50.0 /24
  • Router C ↔ Router A: 192.168.60.0 /24

4. Comandos de Interfaz y Enrutamiento Estático

A continuación, se muestra la configuración clave para el Router A como ejemplo. Se debe seguir la misma lógica para los Routers B y C.

Paso 1: Asignar IPs a las interfaces

! Configurando interfaz hacia la LAN (Red A)
RouterA(config)# interface gigabitEthernet 0/0
RouterA(config-if)# ip address 192.168.10.1 255.255.255.0
RouterA(config-if)# no shutdown

! Configurando interfaz hacia Router B
RouterA(config)# interface gigabitEthernet 0/1
RouterA(config-if)# ip address 192.168.40.1 255.255.255.0
RouterA(config-if)# no shutdown

Paso 2: Crear las Rutas Estáticas

La sintaxis es: ip route [Red de Destino] [Máscara] [IP del vecino por donde pasar]

! Enseñándole a Router A cómo llegar a la Red B
RouterA(config)# ip route 192.168.20.0 255.255.255.0 192.168.40.2

! Enseñándole a Router A cómo llegar a la Red C
RouterA(config)# ip route 192.168.30.0 255.255.255.0 192.168.60.1
